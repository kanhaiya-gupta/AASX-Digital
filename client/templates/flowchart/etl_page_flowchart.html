System Architecture Description

ETL System Architecture

The ETL system follows a modern microservices architecture with clear separation between frontend UI, backend API, processing engine, and data storage layers.

Frontend Layer

Modern web interface built with Bootstrap and JavaScript, providing intuitive project management, file upload, and progress monitoring capabilities.

Backend API

FastAPI-based REST API handling project CRUD operations, file management, ETL configuration, and real-time progress tracking.

Processing Engine

Hybrid Python/.NET processing engine with specialized components for AASX parsing, data transformation, and multi-format output generation.

Data Storage

Multi-layered storage architecture including in-memory project/file tracking, SQLite databases, vector embeddings, and graph databases.

Progress Tracking

Real-time progress monitoring system with status updates, error handling, and user notifications throughout the ETL process.

Error Handling

Comprehensive error management with detailed logging, user-friendly error messages, and retry mechanisms for failed operations.

Data Flow Description

Data Processing Pipeline

The ETL data flow transforms raw AASX files through extraction, transformation, and loading phases, creating multiple interconnected data formats for comprehensive analysis.

Extract Phase

Parses AASX ZIP archives to extract XML specifications, binary content, and metadata, creating structured representations of asset shells, submodels, and relationships.

Transform Phase

Normalizes data formats, maps properties, builds relationships, processes documents, and enriches metadata to create consistent, queryable data structures.

Load Phase

Stores processed data in multiple specialized formats: SQLite for structured data, Neo4j for graph relationships, Qdrant for vector embeddings, and file exports for portability.

Quality Control

Validates data quality, generates comprehensive reports, and identifies issues that need attention, ensuring reliable data for downstream analysis.

Data Organization

Organizes outputs by project with cross-project relationships, creating searchable metadata indexes and maintaining processing history for audit trails.

Analysis Ready

Produces analysis-ready data in multiple formats supporting AI/RAG systems, graph analytics, traditional database queries, and export capabilities.
