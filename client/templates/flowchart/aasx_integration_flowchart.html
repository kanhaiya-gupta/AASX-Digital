AASX Auto Integration Flow Description

This flowchart illustrates the automated process for discovering ETL-processed AASX files and registering them as digital twins in the Twin Registry. It shows how the system scans project directories, extracts twin information, and provides a seamless integration between AASX data and digital twin management.

ETL Output Discovery

Scans the ETL output directory structure to discover processed AASX files organized by project, enabling automatic twin registration from existing data.

Twin Information Extraction

Extracts twin metadata, asset information, and project context from SQLite databases and export files generated by the ETL pipeline.

Twin Registration

Registers discovered twins in the Twin Registry database with proper validation, sync status tracking, and AASX file associations.

API & UI Integration

Provides REST API endpoints and user interface for project discovery, file browsing, twin registration, and sync status monitoring.

Process Steps Breakdown
Discovery Phase
  • Scan ETL output directories
  • Identify project structure
  • List available files per project
  • Detect file formats (JSON/YAML/CSV)
Processing Phase
  • Extract twin metadata
  • Parse asset information
  • Validate twin data
  • Check for duplicates
Registration Phase
  • Register twins in database
  • Update sync timestamps
  • Link AASX source files
  • Store project associations
Integration Phase
  • Expose REST API endpoints
  • Provide UI dashboard
  • Enable user interactions
  • Monitor sync status
