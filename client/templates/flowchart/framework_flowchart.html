Flow Description

Framework Flow Description

The AASX Digital Twin Analytics Framework follows a comprehensive processing flow that transforms AASX files into actionable insights through multiple stages of data processing, analysis, and management.

1. AASX Package Explorer

Professional tool for creating, editing, and saving AASX files. Users can design digital twin models, define asset relationships, and export standardized AASX packages for processing.

2. AASX ETL Pipeline

Processes AASX files from various infrastructure sources (additive manufacturing, hydrogen stations, smart grids, etc.) through extraction, transformation, and loading phases.

3. Knowledge Graph Construction

Builds Neo4j-based knowledge graphs from structured data and metadata, creating relationship networks between assets, properties, and components.

4. AI/RAG Analysis

Provides intelligent analysis through OpenAI integration, combining semantic search and graph queries to generate comprehensive responses to natural language questions.

5. Digital Twin Management

Manages digital twin registrations and monitors asset health through the twin registry and certificate management systems.

6. Analytics & Insights

Generates actionable insights through the analytics dashboard, combining data from all sources to provide comprehensive digital twin analytics.

Goal: This framework provides a complete digital twin lifecycle from creation (AASX Package Explorer) to analysis (AI/RAG) and management (Twin Registry), transforming complex AASX data into actionable insights through automated ETL processing, intelligent AI analysis, and comprehensive digital twin management, supporting critical infrastructure operations and decision-making.

ETL Pipeline Description

ETL Pipeline Flow Description

The ETL (Extract, Transform, Load) pipeline processes AASX files through three main phases, transforming raw AASX data into structured, queryable formats for analysis and insights.

Extract Phase

Validates and extracts content from AASX files, parsing XML/JSON structures and extracting assets, submodels, documents, and metadata.

Transform Phase

Normalizes data formats, processes documents, enriches metadata, and resolves entity relationships across the extracted data.

Load Phase

Stores processed data in multiple formats: SQLite databases, vector embeddings, graph relationships, and export files organized by project.

Quality Control

Validates data quality, generates error reports, and provides retry mechanisms for failed processing steps.

AASX Auto Integration Description

AASX Auto Integration Flow Description

This flowchart illustrates the automated process for discovering ETL-processed AASX files and registering them as digital twins in the Twin Registry. It shows how the system scans project directories, extracts twin information, and provides a seamless integration between AASX data and digital twin management.

ETL Output Discovery

Scans the ETL output directory structure to discover processed AASX files organized by project, enabling automatic twin registration from existing data.

Twin Information Extraction

Extracts twin metadata, asset information, and project context from SQLite databases and export files generated by the ETL pipeline.

Twin Registration

Registers discovered twins in the Twin Registry database with proper validation, sync status tracking, and AASX file associations.

API & UI Integration

Provides REST API endpoints and user interface for project discovery, file browsing, twin registration, and sync status monitoring.
