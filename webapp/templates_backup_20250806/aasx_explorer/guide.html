User Guide

Complete guide to creating, editing, and managing AASX packages with the Eclipse AASX Package Explorer

Quick Start Guide

Getting Started
  1. Download and install AASX Package Explorer
  2. Launch the application
  3. Create a new AASX package or open existing file
  4. Add your first asset
  5. Configure submodels and properties
  6. Validate and save your package
System Requirements
  • OS: Windows 10/11
  • .NET: .NET Framework 4.7.2 or later
  • RAM: 4GB minimum, 8GB recommended
  • Storage: 500MB free space
  • Display: 1024x768 minimum resolution

Step-by-Step Tutorial

Learn how to create your first AASX package

1
Create New AASX Package

Start by creating a new AASX package to represent your asset.

  1. Click "File" → "New" → "AASX Package"
  2. Enter a name for your package
  3. Choose a save location
  4. Click "Create" to initialize the package
Tip: Use descriptive names that reflect your asset type
2
Add Asset Administration Shell

Create the main Asset Administration Shell for your asset.

  1. Right-click on "Asset Administration Shells"
  2. Select "Add Asset Administration Shell"
  3. Enter asset identification details
  4. Configure asset type and description
Tip: Use unique identifiers for your assets
3
Create Submodels

Add submodels to organize different aspects of your asset.

  1. Right-click on your AAS
  2. Select "Add Submodel"
  3. Choose submodel type (Technical Data, Operational Data, etc.)
  4. Configure submodel properties
Tip: Use standard submodel templates when available
4
Add Properties and Elements

Define properties and other elements within your submodels.

  1. Select a submodel
  2. Right-click and choose "Add Property"
  3. Define property name, type, and value
  4. Add additional elements as needed
Tip: Use appropriate data types for your properties
5
Validate Your Package

Ensure your AASX package complies with the specification.

  1. Click "Tools" → "Validate Package"
  2. Review validation results
  3. Fix any errors or warnings
  4. Re-validate until all issues are resolved
Tip: Address all validation errors before proceeding
6
Save and Export

Save your AASX package and export for use in other systems.

  1. Click "File" → "Save" to save your work
  2. Use "File" → "Export" for additional formats
  3. Choose export format (JSON, XML, etc.)
  4. Select destination and export
Tip: Keep backup copies of your AASX files

Best Practices

Naming Conventions
  • Use descriptive, consistent names
  • Follow industry standards when available
  • Avoid special characters in identifiers
  • Use camelCase for property names
Organization
  • Group related properties in submodels
  • Use standard submodel templates
  • Maintain logical hierarchy
  • Document your structure
Data Quality
  • Use appropriate data types
  • Include units of measurement
  • Validate data regularly
  • Keep data up to date
Integration
  • Plan for system integration
  • Use standard identifiers
  • Consider scalability
  • Test with target systems
