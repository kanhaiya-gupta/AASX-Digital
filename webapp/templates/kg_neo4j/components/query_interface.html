Get All Nodes
MATCH (n) RETURN n LIMIT 10 Returns first 10 nodes in the database
Get All Relationships
MATCH (a)-[r]->(b) RETURN a, r, b LIMIT 10 Returns first 10 relationships with their connected nodes
Get Valid Relationships (Recommended)
MATCH (n)-[r]->(m) WHERE n.id IS NOT NULL AND m.id IS NOT NULL RETURN n, r, m LIMIT 10 Returns relationships only between nodes with valid IDs (more reliable)
Count Nodes by Type
MATCH (n) RETURN labels(n) as NodeType, count(n) as Count ORDER BY Count DESC Shows distribution of nodes by their labels
Database Statistics
MATCH (n) RETURN count(n) as TotalNodes UNION MATCH ()-[r]->() RETURN count(r) as TotalRelationships Shows total count of nodes and relationships in the database

Find Submodels
MATCH (n:Submodel) RETURN n.idShort as Submodel, n.description as Description Finds all Submodel nodes with their descriptions
Find Properties
MATCH (n:Property) RETURN n.idShort as Property, n.value as Value LIMIT 20 Shows Property nodes with their values
Submodel References
MATCH (a)-[r:submodel_reference]->(b) RETURN a.idShort as From, b.idShort as To Shows submodel reference relationships

Nodes from Specific File
MATCH (n) WHERE n.file_id = 'your-file-id-here' RETURN n LIMIT 20 Replace 'your-file-id-here' with actual file ID
File Statistics
MATCH (n) WHERE n.file_id = 'your-file-id-here' RETURN labels(n) as Type, count(n) as Count Shows node count by type for a specific file
Valid Relationships from File
MATCH (n)-[r]->(m) WHERE n.file_id = 'your-file-id-here' AND n.id IS NOT NULL AND m.id IS NOT NULL RETURN n, r, m LIMIT 20 Get relationships from a specific file with valid node IDs
All Files in Database
MATCH (n) WHERE n.file_id IS NOT NULL RETURN DISTINCT n.file_id as FileID, count(n) as NodeCount ORDER BY NodeCount DESC Lists all files in the database with their node counts

Find Connected Components
MATCH path = (start)-[*]-(end) WHERE start.idShort = 'YourNode' RETURN path LIMIT 5 Finds all nodes connected to a specific node (up to 5 paths)
Filter by Property Value
MATCH (n) WHERE n.value CONTAINS 'specific_value' RETURN n Finds nodes with specific value in their properties
Find Nodes Without IDs
MATCH (n) WHERE n.id IS NULL RETURN n LIMIT 10 Useful for debugging - finds nodes that might cause issues
Find Orphaned Relationships
MATCH (n)-[r]->(m) WHERE n.id IS NULL OR m.id IS NULL RETURN n, r, m LIMIT 10 Finds relationships connected to nodes without proper IDs
Delete All Data (Dangerous!)
MATCH (n) DETACH DELETE n ⚠️ This will delete ALL nodes and relationships!
